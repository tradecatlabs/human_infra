# Dams Levees Flood Control Infrastructure Safety

`dams-levees-flood-control-infrastructure-safety/` 研究水坝、堤防、防洪工程、闸门、排涝系统和洪水控制基础设施的安全、维护、失效风险和公共治理。

> 核心问题：洪水风险不只来自降雨和河流，也来自防洪基础设施能否被设计、维护、监测、检查和治理。水坝或堤防失效会把低概率工程风险瞬间变成生命、住房、交通、能源和公共服务的系统性中断。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 主体依赖居住地、医院、道路、能源和公共服务不被洪水突然摧毁
  -> 许多社区的洪水安全依赖水坝、堤防、闸门、泵站和排涝系统
  -> 老化、维护不足、极端降雨、溢流、渗漏、结构退化和治理失败会改变尾部风险
  -> 防洪基础设施安全失败会在水文预警和灾后恢复之前击穿主体持续性
```

## 关注对象

- 水坝、堤防、防洪墙、闸门、泵站、排涝系统、溢洪道、检查、维护、风险分类和安全治理。
- 工程资产老化、极端降雨适应、下游暴露、预警联动、应急行动计划和风险沟通边界。
- 与 `water-resources-hydrology-flood-drought-management/` 的边界：水资源域关注水文和流域风险；本域关注防洪工程资产与失效风险。
- 与 `disaster-recovery-relief-continuity/` 的边界：灾后恢复域关注冲击后的救济；本域关注工程失效前的安全治理。

## Human Infra 模型链路

```text
防洪工程资产安全治理
  -> 水坝、堤防、闸门、泵站和排涝系统状态可被检查和维护
  -> 洪水尾部风险、下游暴露、预警联动和失效场景进入公共治理
  -> 突发淹没、住房损失、交通中断、医院中断和长期恢复成本下降
  -> 主体获得被工程系统保护而不是被工程失效放大的水风险边界
```

## 非目标

- 不提供水坝、堤防、防洪墙、泵站、闸门、排涝系统或水库调度的工程设计、检修、操作或加固建议。
- 不提供实时洪水行动、撤离路线、闸门操作、溃坝模拟、敏感设施位置、破坏方法或规避监管信息。
- 不替代水利、应急、防洪、工程、地方政府或设施所有者的官方信息。
