# Possibility Space Control Rubric

## 目标

给研究域分级时，只问一个核心问题：

```text
如果这个域成立或失效，它能在多大程度上改写主体未来可能性空间？
```

不要先问：

```text
现在能不能做？
证据成熟不成熟？
有没有产品？
近期能不能落地？
```

这些是证据成熟度和执行优先级，不是根本性层级。

## 判定顺序

1. 先看是否直接触及主体持续性边界：寿命、死亡、时间、身份连续性、未来抵达。
2. 再看是否维护产生可能性的生命载体：脑、身体、器官、衰老、损伤、急性生命失败。
3. 再看是否放大主体生成路径的能力：认知、学习、注意力、记忆、AI、工具、知识。
4. 再看是否把潜在路径转成真实路径：医疗、科研、数据、转化、制度、资金、协作。
5. 再看是否提供外部生态承载：能源、水、食物、通信、治理、经济、基础设施、灾害恢复。
6. 最后落到具体局部问题：单病种、单流程、单服务、专项风险、局部摩擦。

## 层级定义

### C0 可能性原点层

主体持续性本身。它不是普通研究域，而是所有研究域的价值标尺。

判定句：

```text
没有持续主体，可能性空间归零。
```

### C1 可能性边界改写层

直接改变主体能否继续、能经历多少固有时间、能否跨越死亡边界、能否抵达未来、是否仍保持主体连续性。

典型对象：

- 有效永生
- 长寿逃逸速度
- 未来等待 / 时间差分路径
- 冷冻保存 / 生物停滞
- 脑保存 / connectomics / emulation
- 去具身中枢生命系统
- 记忆、人格和身份连续性

### C2 可能性源体维护层

维护主体产生可能性的生命载体。它不一定改写终局，但决定主体是否提前坍塌。

典型对象：

- 脑、神经系统、意识状态
- 身体、氧合、循环、器官系统
- 衰老机制、癌症、免疫、感染、再生医学
- 细胞重编程、基因组稳定、线粒体、蛋白稳态
- 急救、ICU、器官支持、创伤、脓毒症

### C3 可能性生成引擎层

放大主体创造、选择、学习、修正和利用未来技术的能力。

典型对象：

- 认知、学习、注意力、记忆
- AI 增强、工具使用、自动化
- 个人知识管理、认知卸载
- 教育、技能形成、反馈闭环

### C4 可能性转换通道层

把科学、技术、资源、制度从“存在”转成主体真实可用路径。

典型对象：

- 医疗可及、临床转化、试验治理
- 数据互操作、隐私计算、知识图谱、模型审计
- 科研基础设施、开放科学、技术转移
- 法律身份、公共服务入口、行政负担削减

### C5 可能性生态承载层

承载主体展开未来的外部世界条件。

典型对象：

- 能源、食物、水、住房、交通、通信
- 宏观经济、治理、反腐败、司法、公共安全
- 灾害恢复、气候韧性、供应链、关键基础设施

### C6 局部可能性解锁层

解除具体场景中的局部阻塞。它可以很重要、很紧急，但通常不直接改写总边界。

典型对象：

- 具体疾病执行域
- 具体服务流程
- 职业、家庭、社区、学校、法院、福利和消费流程
- 专项设备、用品、申诉、续期、导航和风险点

## 边界案例

| 情况 | 优先判定 |
| --- | --- |
| 远期但改写主体时间边界 | C1 |
| 近期可行但只是单流程优化 | C6 |
| 医疗服务可及 | C4 |
| 某个具体疾病的筛查/随访 | C6，除非它是系统级生命载体域 |
| AI 作为主体能力增强 | C3 |
| AI 作为基础设施服务、数据中心或治理系统 | C4 或 C5 |
| 研究工具、论文、数据、标准 | C4 |
| 能源、水、食物、住房、通信 | C5 |

## 输出字段

`classification.tsv` 使用以下字段：

```text
domain
tier
tier_name
control_axis
rationale
review_status
```

`review_status` 默认是 `heuristic-v0.1`，表示初筛结果，需要后续人工复核。
