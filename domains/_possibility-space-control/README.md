# Possibility Space Control

`_possibility-space-control/` 是 `domains/` 的元索引目录，不是普通研究域。它用于按照“可能性空间控制力”给全部研究域分级，并维护研究域在 C1-C6 物理目录中的真实落点。

核心标尺：

```text
研究域的根本性
= 它一旦成立，能够多深、多大、多长期地改写主体未来可能性空间
```

这个目录不按近期可行性、证据成熟度、工程成本或商业化时间排序。远期、困难、不可立即落地的路径，如果直接改写主体持续性边界，也可以位于最高层。

## 分层

| 层级 | 物理目录 | 名称 | 判定 |
| --- | --- | --- | --- |
| C0 | 无普通研究域目录 | 可能性原点层 | 主体持续性本身；这是价值核，不是普通研究域。 |
| C1 | `../c1-boundary-rewriting/` | 可能性边界改写层 | 直接改写寿命、死亡、时间、身份连续性和未来抵达边界。 |
| C2 | `../c2-source-maintenance/` | 可能性源体维护层 | 维护产生可能性的身体、脑、生命系统和衰老/损伤控制。 |
| C3 | `../c3-generation-engine/` | 可能性生成引擎层 | 提升主体创造新路径的认知、学习、AI、工具和能力复利。 |
| C4 | `../c4-conversion-channel/` | 可能性转换通道层 | 把潜在可能性转成真实路径的研究、医疗、数据、制度和转化接口。 |
| C5 | `../c5-ecological-substrate/` | 可能性生态承载层 | 承载主体展开未来的能源、食物、水、治理、经济、通信和基础设施环境。 |
| C6 | `../c6-local-unlocking/` | 局部可能性解锁层 | 解除具体疾病、流程、服务、职业、家庭、社区和专项风险中的局部阻塞。 |

## 文件

- `rubric.md`：分级规则、判定顺序和边界案例。
- `classification.tsv`：全部正式研究域的初版分级表，包含每个研究域的 `physical_path`。
- `summary.md`：按层级统计和人工复核重点。
- `AGENTS.md`：维护边界和更新规则。

## 解释边界

- C1 不是“最可行”，而是“最能改写边界”。
- C2 不是“次要生命”，而是“载体维护”；它决定主体是否提前坍塌。
- C3 不是普通效率优化，而是主体生成新路径的能力系统。
- C4/C5 是可能性从潜在变成现实的通道和环境。
- C6 不等于不重要；它只是相对更偏具体阻塞和边界内优化。

## 更新规则

新增研究域后，必须放入对应 C1-C6 物理目录，并同步更新 `classification.tsv`、`summary.md`，说明它的根本性层级和 `physical_path`，而不是只说明证据或可行性。
