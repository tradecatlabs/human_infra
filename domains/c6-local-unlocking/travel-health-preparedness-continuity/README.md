## 研究推进骨架

### 最小问题集

- 界定局部阻塞：某一疾病、流程、服务、职业、家庭、社区或专项风险中的具体失败点。
- 说明受影响任务：它阻碍了主体完成什么行动、服务、照护、转换或恢复过程。
- 记录可测改善：等待时间、错误率、成本、可及性、完成率、伤害率或恢复速度。
- 明确升级边界：何时应转交 C2-C5 的上层生命、能力、转化或基础设施域。
- 避免把局部改良包装成根本性解决方案，必须标注适用范围和外部依赖。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

本域研究旅行健康风险、目的地健康信息、疫苗与预防资料、慢病旅行准备、药品供应、医疗保险和境外医疗可及如何影响主体离家后的健康与任务连续性。它关注的是旅行健康准备作为风险治理层，而不是个人医学建议或旅行门诊替代品。

## 研究对象

- CDC Travelers' Health、Yellow Book、目的地健康通知、疫苗与预防资料、旅行医学和出行前健康准备。
- 慢病、药品、医疗设备、保险、医疗记录、过敏、免疫状态、孕产、儿童和老年旅行健康变量。
- 传染病、食水安全、热暴露、高海拔、动物暴露、事故、精神健康和医疗可及性风险。
- 旅行健康资料如何连接航空、住宿、行李、领事协助、跨境入境和返程恢复。

## 价值链路

```text
离家移动会改变主体暴露环境和医疗可及性
  -> 目的地疾病、药物供应、医疗系统、气候和活动风险改变健康状态转移函数
  -> 旅行健康准备把风险识别、预防资料、药品设备和医疗求助链提前纳入计划
  -> 若准备失败，普通行程中断会扩散为疾病、治疗中断、滞留和未来选择权损失
  -> 旅行健康准备连续性把短期旅行建模为健康寿命与有效时间的暴露管理问题
```

## 关键问题

- 旅行如何改变感染、慢病、药物、热暴露、事故和医疗可及性风险？
- 官方旅行健康资料如何进入 Human Infra 的风险、状态和恢复模型？
- 药品、设备、过敏、免疫和医疗记录如何与行李、安检、住宿和边境衔接？
- 如何避免把旅行健康资料误写成个人诊疗、疫苗、处方或自我治疗建议？

## 非目标

- 不提供个人诊断、疫苗建议、处方、用药、旅行医学门诊替代、疫情判断、保险购买、医疗转运或目的地安全建议。
- 不判断个人是否适合旅行、是否需要某疫苗、是否可携带某药、是否应改变治疗方案。
- 不收集健康记录、疫苗记录、药品清单、诊断、保险信息、目的地行程或个人风险画像。
