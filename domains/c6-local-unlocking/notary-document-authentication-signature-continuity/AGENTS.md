# 公证、文件认证与签名连续性目录说明

## 目录结构

```text
notary-document-authentication-signature-continuity/
├── AGENTS.md
└── README.md
```

## 文件职责

- `README.md`：定义公证、签名、电子签名、文件认证和跨机构承认如何支撑授权与记录连续性。
- `AGENTS.md`：说明本域维护边界和禁止用途。

## 维护规则

- 只沉淀文件认证机制、签名证据链、可验证性和跨机构承认变量，不写个体文件办理流程。
- 涉及美国制度时优先使用 State Department、州 secretary of state、官方 notary 入口、NIST 和机构官方要求。
- 不把本域写成公证、apostille、电子签名、合同、移民、学校、法院、继承或个案法律建议。
