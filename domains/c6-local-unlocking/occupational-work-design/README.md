### Human Infra 追问

这个域解除的是哪一个具体疾病、流程、服务、职业、家庭、社区或专项风险中的局部阻塞？

```text
研究域对象
  -> 影响变量 / 中间机制
  -> 改变主体状态或外部条件
  -> 改变风险、能力、时间成本或可达性
  -> 改变有效寿命、有效时间或未来选择权
```

## 研究推进骨架

### 最小问题集

- 界定局部阻塞：某一疾病、流程、服务、职业、家庭、社区或专项风险中的具体失败点。
- 说明受影响任务：它阻碍了主体完成什么行动、服务、照护、转换或恢复过程。
- 记录可测改善：等待时间、错误率、成本、可及性、完成率、伤害率或恢复速度。
- 明确升级边界：何时应转交 C2-C5 的上层生命、能力、转化或基础设施域。
- 避免把局部改良包装成根本性解决方案，必须标注适用范围和外部依赖。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

Occupational Work Design 负责整理工作组织、职业安全、人体工学、工作压力、劳动时间、职业暴露、技能转型和工作可持续性如何进入主体持续性模型。

核心问题：

> 工作是时间、收入、身份、风险暴露、学习机会和社会连接的交汇点；坏的工作设计会消耗身体、注意力、心理和寿命窗口，好的工作设计能延长有效行动能力。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 主体需要通过工作或任务系统获得资源、意义、协作和学习机会
  -> 工作结构同时决定暴露风险、压力、休息、收入、注意力和技能积累
  -> 职业伤害、过劳、失业、不可持续工作或自动化冲击会截断主体行动
  -> 因此职业与工作设计是主体持续性的任务环境域
```

## 关注对象

- 职业安全、工作组织、人体工学、工时、轮班、压力、倦怠、职业暴露和工作-恢复边界。
- 就业质量、技能转型、自动化影响、远程工作、照护冲突、工作自主性和可持续职业路径。
- 工作如何进入财务韧性、心理稳定、时间分配、身体损耗、学习能力和社会连接模型。
- 与 `resource-social-infra/` 的边界：本域关注任务/工作环境；资源社会基础设施关注更广义的社会资源与服务底座。

## Human Infra 模型链路

```text
工作设计因素 T
  -> 改变工时、暴露、压力、人体工学、自主性、收入和学习机会变量 X
  -> 改变主体任务环境状态 S
  -> 改变职业伤害、过劳、倦怠、收入断裂、技能过时和恢复不足风险 λ(t)
  -> 影响有效时间、健康寿命、能力积累和未来选择权
```

## 非目标

- 不提供个体职业规划、劳动争议、法律、工伤、移民、薪酬谈判或雇佣建议。
- 不提供绕过安全规范、压榨劳动、监控员工或规避监管的方法。
- 不把生产率最大化等同于主体持续性；必须审查恢复、权利、安全和长期能力。
