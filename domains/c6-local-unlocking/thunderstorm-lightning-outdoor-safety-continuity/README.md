## 研究推进骨架

### 最小问题集

- 界定局部阻塞：某一疾病、流程、服务、职业、家庭、社区或专项风险中的具体失败点。
- 说明受影响任务：它阻碍了主体完成什么行动、服务、照护、转换或恢复过程。
- 记录可测改善：等待时间、错误率、成本、可及性、完成率、伤害率或恢复速度。
- 明确升级边界：何时应转交 C2-C5 的上层生命、能力、转化或基础设施域。
- 避免把局部改良包装成根本性解决方案，必须标注适用范围和外部依赖。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

`thunderstorm-lightning-outdoor-safety-continuity/` 研究雷暴、闪电、强风、冰雹、户外活动、工作暴露和停电树倒风险如何影响主体持续性。

> 核心问题：雷暴和闪电常被低估，但它们通过户外暴露、树木/电线、短时强风、冰雹和突然停电直接切断日常任务窗口。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 主体日常行动大量发生在户外、交通、学校、工作和娱乐场景
  -> 雷暴闪电会在短时间内制造电击、坠物、交通、电力和通信风险
  -> 若 watch/warning、避险空间、活动暂停和恢复判断失败，日常任务会被急性 hazard 截断
  -> 因而雷暴闪电户外安全是低频高后果日常风险执行域
```

## 关注对象

- 雷暴、闪电、强风、冰雹、倒树、停电、户外工作、学校体育、露营、船艇和大型活动。
- 警报理解、安全室内空间、活动暂停、复工复赛、通信失灵和伤害后急救入口。
- 与 `weather-climate-observation-forecasting/` 的边界：该域关注观测与预报；本域关注雷暴警报后的户外行动治理。

## Human Infra 模型链路

```text
雷暴闪电户外安全系统 T
  -> 改变警报接收、活动暂停、避险空间、户外工作安排、树电线暴露和通信变量 X
  -> 改变主体在短时强对流窗口中的暴露状态 S
  -> 改变雷击、创伤、停电、交通事故和任务中断风险 λ(t)
  -> 影响急性生存、有效时间、工作学校连续性和恢复负担
```

## 非目标

- 不提供实时雷暴判断、户外活动许可、赛事/学校停开决策、电线处理、树木处理、船艇航行或医疗急救建议。
- 不替代 NWS、地方应急管理、学校、雇主、活动组织者、公用事业或公共安全机构。
- 不鼓励户外冒险、靠近倒线、躲在树下、风暴观测或无授权现场处理。
