## 研究推进骨架

### 最小问题集

- 界定外部承载条件：能源、水、食物、住房、通信、交通、治理、经济或公共基础设施。
- 说明它如何改变主体展开未来所需的稳定性、冗余、可获得性和安全边界。
- 记录基础设施依赖：供应链、地理环境、制度、资金、维护能力和公共服务。
- 识别冲击场景：灾害、战争、断供、污染、气候、系统失灵或社会支持崩溃。
- 建立韧性指标：可用率、恢复时间、冗余、成本、覆盖范围、可负担性和风险暴露。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

`undersea-cable-internet-backbone-continuity/` 研究海底光缆、登陆站、跨境数据传输、路由冗余、维修船、物理安全和国际治理如何支撑全球互联网、云服务、科研协作和数字主体连续性。

> 核心问题：云、AI、远程医疗、跨境协作和数字记忆都依赖全球骨干网络；海底光缆断裂或登陆站风险会把数字基础设施从全球系统压回局部孤岛。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 需要云服务、AI、数据同步、科研协作和跨境通信长期可达
  -> 全球互联网高度依赖海底光缆、登陆站、维修和路由冗余
  -> 光缆中断或地缘风险会导致连接延迟、数据不可达和服务退化
  -> 因此海底光缆骨干是数字主体连续性的全球连接约束域
```

## 关注对象

- Submarine cable、landing station、route diversity、repair vessel、cable cut、cross-border data flow、physical security、licensing 和 resilience。
- 云服务跨区、科研数据传输、跨境医疗沟通、支付、通信、灾害恢复和内容分发的骨干依赖。
- 与 `telecommunications-network-resilience-continuity/` 的边界：电信韧性域是总域；本域专门拆出跨洋骨干和登陆站风险。
- 与 `cloud-service-exit-interoperability-continuity/` 的关系：云退出关注服务迁移；本域关注网络路径可达。

## Human Infra 模型链路

```text
海底光缆骨干 T
  -> 改变跨境带宽、路由冗余、维修时长、登陆站风险和延迟变量 X
  -> 改变云服务、AI 工具、科研协作、支付和通信可达状态 S
  -> 改变连接中断、区域隔离、数据同步失败和服务退化风险 λ(t)
  -> 影响主体数字记忆、远程协作、资源访问和全球未来选择权
```

## 非目标

- 不提供海底光缆位置敏感细节、破坏方式、安保规避、网络攻击或监控方法。
- 不输出运营商选择、路由采购、投资或地缘政治预测建议。
- 不替代电信网络、云服务或网络安全域。
