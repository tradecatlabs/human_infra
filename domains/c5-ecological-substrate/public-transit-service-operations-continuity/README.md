### Human Infra 追问

这个域如何承载主体展开未来的外部世界条件，例如能源、水、食物、住房、通信、治理、经济和基础设施？

```text
研究域对象
  -> 影响变量 / 中间机制
  -> 改变主体状态或外部条件
  -> 改变风险、能力、时间成本或可达性
  -> 改变有效寿命、有效时间或未来选择权
```

## 研究推进骨架

### 最小问题集

- 界定外部承载条件：能源、水、食物、住房、通信、交通、治理、经济或公共基础设施。
- 说明它如何改变主体展开未来所需的稳定性、冗余、可获得性和安全边界。
- 记录基础设施依赖：供应链、地理环境、制度、资金、维护能力和公共服务。
- 识别冲击场景：灾害、战争、断供、污染、气候、系统失灵或社会支持崩溃。
- 建立韧性指标：可用率、恢复时间、冗余、成本、覆盖范围、可负担性和风险暴露。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

`public-transit-service-operations-continuity/` 研究公交、轨道、地铁、轻轨、轮渡、无障碍交通、班次可靠性、运营资金、司机/运维人力和服务恢复如何支撑主体持续性。

> 核心问题：交通可达不只是“有路”。大量主体依赖公共交通到达医疗、工作、教育、食物、照护和社会关系。公交运营失败会把资源存在转化为资源不可达。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 主体需要稳定到达医疗、工作、学习、食物、照护、公共服务和关系网络
  -> 公共交通依赖班次、车辆、司机、维修、站点、无障碍、资金、调度和信息服务
  -> 运营削减、延误、司机短缺、设备故障、灾害和财政压力会造成服务断点
  -> 公交断点会增加缺诊、失业、孤立、时间贫困和撤离不可达
```

## 关注对象

- 公交、轨道交通、地铁、轻轨、轮渡、需求响应交通、paratransit、班次可靠性、服务覆盖、无障碍运营、乘客信息、资产状态和运营资金。
- 与 `transportation-access-mobility/` 的边界：交通接入域关注日常移动和资源可达；本域关注公共交通系统自身的运营连续性、服务水平和恢复能力。
- 与 `road-traffic-injury-prevention-safe-mobility/` 的边界：道路安全域关注伤害预防；本域关注公共交通服务是否持续可用。

## Human Infra 模型链路

```text
公共交通运营连续性
  -> 班次、覆盖、无障碍、车辆、站点、司机、维护和实时信息稳定运行
  -> 医疗、就业、教育、食品、照护和公共服务可达性提升
  -> 缺诊、迟到、时间贫困、孤立和灾害撤离断点下降
  -> 主体获得低成本、可预测、低摩擦的移动生命线
```

## 非目标

- 不提供实时路线、逃票、绕闸、破坏、设施弱点、司机调度、车辆维修、运营控制、安保规避或危险乘车建议。
- 不提供交通投资、项目采购、票价争议、法律索赔、个体出行规划或实时灾害撤离方案。
- 不替代当地 transit agency、FTA、交通管理、应急管理和官方服务公告。
