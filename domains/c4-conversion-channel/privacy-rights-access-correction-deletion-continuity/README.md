Privacy Rights Access Correction Deletion Continuity 关注个人对自身数据的访问、更正、删除、限制处理、可携带、申诉和执行权利如何让主体在数据化社会中保持可纠错、可退出和可控制。

## 对象

- Privacy rights、data subject access request、correction、deletion、appeal、verification、authorized agent 和 response timeline。
- 数据错误、身份混淆、旧资料、敏感资料外泄、不可解释拒绝和跨机构数据同步失败。
- 消费者、患者、学生、劳动者、租户、账户持有人、儿童和照护代理人的数据权利入口。

## Human Infra 价值

```text
主体持续性需要数据系统承认主体的纠错权
-> 外部系统持续记录主体身份、行为、资格、风险和偏好
-> 数据错误或无法访问会让主体失去解释、纠正和退出机会
-> 访问、更正、删除、申诉和授权代理机制把数据系统变成可审查接口
-> 主体在金融、住房、医疗、就业、教育和公共服务中的连续性更少被错误资料击穿
```

## 边界

- 本域不提供法律建议、隐私权适用判断、请求信代写、身份验证材料准备、投诉策略或平台操作教程。
- 不判断某机构是否必须删除、更正、披露或停止处理某项个人数据。
- 不提供绕过平台、伪造授权、隐藏身份、抓取数据、批量投诉、骚扰企业或攻击隐私门户的方法。
- 只整理公开资料、权利类型、请求流程抽象、失败模式、证据边界和禁止用途。
