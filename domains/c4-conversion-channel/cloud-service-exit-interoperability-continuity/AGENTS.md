# cloud-service-exit-interoperability-continuity 目录说明

本目录承载云服务退出与互操作连续性研究域，关注云服务锁定、导出和迁移如何影响主体资料与工作流持续性。

## 目录结构

```text
cloud-service-exit-interoperability-continuity/
├── AGENTS.md
└── README.md
```

## 文件职责

- `README.md`：定义云服务、SaaS、导出、迁移、互操作、退出和供应商锁定如何进入 Human Infra 主体持续性模型。
- `AGENTS.md`：说明本目录职责、边界和维护规则。

## 维护规则

- 资料优先回到 EU Data Act、Data Transfer Initiative、NIST cloud portability、主流云厂商公开出口文档和监管资料。
- 不写绕过条款、批量抓取、规避封禁、逃避计费、云入侵或私有数据迁移操作。
- 涉及云退出时只讨论变量、标准、治理边界和可审查迁移能力。
- 新增指标必须注明口径：导出完整性、格式开放性、API 可用性、迁移成本、合同退出或服务替代性。
