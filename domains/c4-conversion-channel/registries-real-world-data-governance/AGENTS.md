# registries-real-world-data-governance 目录说明

本目录维护登记系统、真实世界数据和真实世界证据治理。它是技术进入现实照护场景后的持续反馈层。

## 目录结构

```text
registries-real-world-data-governance/
├── AGENTS.md
└── README.md
```

## 职责边界

- `README.md`：定义患者/疾病/产品登记与 RWD/RWE 如何进入 Human Infra 的证据和治理框架。
- 本目录不保存真实世界微数据、患者记录、claims 明细或可识别健康信息。
- 所有资料必须区分登记目的、数据来源、结局定义、质量审计和用途限制。

## 上下游关系

- 上游来自临床照护、患者数据互操作、监管科学、药物警戒和器械安全。
- 下游服务真实世界安全、长期效果、技术采用反馈、监管学习和模型校正。
- 不替代临床试验域；本目录关注试验之外的真实世界反馈基础设施。

## 维护规则

- 不提供越权访问、重新识别、选择性报告、数据治理绕行或监管规避策略。
- 不把真实世界相关性直接写成因果效应。
- 若新增登记案例，必须记录目的、人群、数据源、质量、偏差和治理边界。
