# notification-alert-routing-attention-continuity 目录说明

本目录承载通知、警报路由与注意力连续性研究域，关注关键事件如何到达主体而不吞噬注意力。

## 目录结构

```text
notification-alert-routing-attention-continuity/
├── AGENTS.md
└── README.md
```

## 文件职责

- `README.md`：定义通知权限、推送、优先级、静音、确认和警报路由如何进入 Human Infra 主体持续性模型。
- `AGENTS.md`：说明本目录职责、边界和维护规则。

## 维护规则

- 资料优先回到 W3C Notifications API、Push API、Permissions API、人因工程和通知疲劳研究。
- 不写垃圾推送、骚扰、绕过通知权限、设备追踪、规避平台限制或操纵用户注意力方案。
- 涉及紧急通知时只讨论路由变量、确认机制、误报漏报和治理边界。
- 新增指标必须注明口径：到达率、确认率、误报率、漏报率、静音风险、渠道冗余或通知负荷。
