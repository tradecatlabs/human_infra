```text
研究域对象
```

- 新增内容必须标明来源、适用对象、证据等级和不确定性；AI 总结不能作为事实源。
- 若内容会改变分级、目录位置或上下游关系，先更新 `domains/_possibility-space-control/classification.tsv`。

## 研究推进骨架

### 最小问题集

- 界定本域直接改写的主体持续性边界：寿命、死亡、身份连续性、时间差分或未来抵达能力。
- 说明主体连续性条件：身体、记忆、人格、行动能力或法律身份中哪些必须保持。
- 列出不可越过的中止门槛：退出失败、连续性失败、尾部风险、不可逆损伤或治理失效。
- 说明它如何改变有效寿命、有效时间、未来选择权或可能性空间。
- 明确本域不能被宣传成现实永生、工程可行性或个体行动方案。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

本子域研究对生物神经系统中记忆痕迹进行读取、写入、删除、重构或调制时涉及的高层模型、证据、验证和伦理边界。

一句话定位：

> 记忆编辑不是控制一个人的工具，而是一个必须以主体连续性、知情同意、可验证性、可退出性和尊严为硬边界的高风险研究域。

## 核心对象

- 记忆痕迹：特定经验、语义、情绪关联或行为倾向在神经系统中的可变表征。
- 读写接口：对记忆相关神经动态进行观察、解释或调制的技术边界。
- 人格连续性：干预前后主体是否仍能保持自我叙事、价值、偏好和权利连续。
- 验证系统：确认干预结果、失败模式、副作用和可逆性的证据框架。
- 治理系统：同意、退出、审计、监督、责任和滥用防护。

## 研究问题

- 记忆、情绪、身份和人格之间的边界如何定义？
- 什么算读取、删除、写入、重构、弱化或再巩固干预？
- 如何证明某次干预没有破坏主体连续性？
- 如何区分治疗性干预、增强性干预和控制性操控？
- 如何建立足够强的同意、撤回、审计和责任机制？

## 安全边界

本子域不提供任何可执行的神经操作、人体实验或记忆操控步骤。

禁止方向：

- 未经同意的记忆读取、写入、删除或重构。
- 面向控制、胁迫、审讯、洗脑或剥夺主体性的设计。
- 把高风险神经干预包装成低风险消费功能。
- 用不完整证据宣称能精确操控真实个体记忆。

允许方向：

- 建立概念边界和证据目录。
- 整理公开文献中的高层技术路线和风险类型。
- 设计非操作性的伦理、审计和验证框架。
- 与 `disembodied-cns` 共同维护主体连续性和尊严原则。

## 与 Human Infra 的关系

记忆编辑属于 Human Infra 的认知基础设施极端边界。普通认知基础设施强调笔记、知识系统、反馈和 AI 工具；记忆编辑则触及生物神经系统本身，因此必须拥有更严格的安全、伦理和证据门槛。
