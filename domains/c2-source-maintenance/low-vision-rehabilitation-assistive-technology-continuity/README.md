## 研究推进骨架

### 最小问题集

- 界定生命载体变量：细胞、组织、器官、脑、免疫、代谢、急性风险或衰老损伤。
- 说明影响机制：修复、替换、再生、控制、筛查、预防、维持或风险降低。
- 列出可观察状态：功能指标、生物标志物、疾病终点、失能时间、恢复能力或死亡风险。
- 记录副作用、过度干预、癌变、免疫、长期安全性和尾部风险。
- 区分机制合理性、体外证据、动物证据、人体早期数据、临床终点和长期安全性。

### 变量接口

- 输入变量：本域直接处理的对象、资源、技术、环境、制度或状态。
- 中间机制：变量通过什么因果路径改变主体状态、系统状态或外部条件。
- 状态改变：身体、认知、能力、资源、风险暴露、可及性、时间成本或协作条件如何变化。
- 风险 / 成本函数：死亡风险、失能风险、工程风险、尾部风险、机会成本、维护成本或治理成本如何变化。
- 输出指标：有效寿命、健康寿命、有效时间、主观时间、相对时间、行动能力、恢复能力或未来选择权。

### 证据入口

- 官方 / 原始资料：监管文件、数据库、临床登记、标准、论文原文、项目白皮书或一手报告。
- 综述 / 指南 / 标准：系统综述、领域指南、技术标准、伦理规范和权威机构材料。
- 数据集 / 登记系统：可复核的数据表、代码、实验记录、登记号、版本和采集边界。
- 反例 / 失败案例：负结果、副作用、安全事故、不可复现结果、伦理争议和误用案例。

### 最小产出

- Source Signals：记录候选资料、来源、日期、主张和待核验点。
- Source Cards：提取 claim、变量、机制、证据类型、边界、反证条件和可迁移位置。
- Claim-Evidence Matrix：把每个稳定主张绑定到来源、证据等级、适用范围和不确定性。
- 变量表：列出输入变量、中间变量、状态变量、风险变量、输出指标和可观测代理指标。
- 上下游关系：说明本域依赖哪些更根本域，并向哪些转化、应用或基础设施域输出。

`low-vision-rehabilitation-assistive-technology-continuity/` 负责整理低视力康复、视觉辅助技术、环境适配、训练维护和不可逆视觉损害后的行动连续性。

核心问题：

> 当视觉损害无法完全恢复时，主体持续性的重点从“治好眼睛”转向“重构任务接口”：用康复、工具、环境和训练把剩余视觉转化为可继续行动的能力。

## 先验位置

```text
主体持续性最大化
  -> 主体不只需要器官功能，还需要可完成任务的接口
  -> 不可逆视觉损害会破坏阅读、移动、自理、沟通和环境识别
  -> 低视力康复和辅助技术能把剩余感知、工具和环境重新编排成行动能力
  -> 因此低视力康复是视觉失效后的主体能力重构基础设施
```

## 关注对象

- 低视力、不可逆视觉损害、剩余视觉使用、放大、照明、对比度和替代输入。
- 低视力康复、定向行走、辅助技术、数字无障碍、训练、维护和费用。
- 阅读、药物识别、厨房安全、移动、工作、学习、社交和独立生活。
- 与 `assistive-technology-access/` 的关系：本域聚焦视觉任务重构；辅助技术域负责跨功能产品可及性。

## Human Infra 模型链路

```text
低视力康复 / 辅助技术 / 环境适配 T
  -> 改变剩余视觉利用、阅读接口、空间定向、工具可用性和训练维护变量 X
  -> 改变主体自理、学习、工作、移动和社交参与状态 S
  -> 改变照护依赖、事故、隔离、任务失败和有效时间损耗风险
  -> 增加补偿能力、行动半径和未来选择权
```

## 非目标

- 不提供低视力诊断、设备处方、训练方案、康复动作、个人安全许可或设备购买建议。
- 不把辅助工具写成通用替代视觉的解决方案；必须保留训练、适配、维护、费用和环境依赖。
- 不替代低视力康复、定向行走、无障碍设计或职业康复专业服务。
