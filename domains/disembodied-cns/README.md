# 去具身外部维持型中枢生命系统

本子域研究一种远期生命形态：自然身体和器官依赖被外部生命支持、传感器、控制器、执行器和神经接口替代，仅保留生物脑或中枢神经系统作为主体连续性的核心载体。

推荐名称：

> 去具身外部维持型中枢生命系统

或更技术化地说：

> 外置生命支持与外部执行器耦合的离体神经中枢系统

## 核心对比

现有人类可以称为“具身半自主开放生命系统”：

```text
生物脑
+ 身体器官
+ 肢体执行系统
+ 感觉系统
+ 自主代谢系统
+ 与外界进行物质 / 能量 / 信息交换
```

目标形态可以称为“去具身外部维持型中枢生命系统”：

```text
生物脑 / 中枢神经系统
+ 人工生命支持系统
+ 电信号输入输出接口
+ 外部控制器
+ 外部执行器
+ 传感器反馈
```

一句话：

> 现有人类是“身体作为主体接口”的生命系统；目标形态是“外部系统作为身体代理”的中枢神经系统。

## 接口架构

### 自然身体接口架构

```text
环境
↓
氧气 / 水 / 食物 / 感觉信息
↓
身体接口：肺、胃肠、皮肤、感官
↓
内部器官系统：循环、代谢、免疫、内分泌
↓
中枢神经系统
↓
神经-肌肉系统
↓
肢体行动 / 语言 / 表情
↓
环境反馈
```

### 人工接口化中枢架构

```text
环境
↓
传感器 / 供能系统 / 灌注系统 / 维生系统
↓
外部控制器
↓
神经接口
↓
生物脑 / 中枢神经系统
↓
神经输出信号
↓
外部控制器
↓
机械执行器 / 机器人 / 虚拟环境
↓
传感器反馈
↓
神经接口
```

## 研究对象

本子域关注高层系统建模，不提供实验步骤。

核心问题：

- 生命支持系统如何代理自然器官的代谢、供氧、清除、温控和保护功能？
- 神经接口如何建立可解释、可校验、低伤害的输入输出闭环？
- 外部控制器如何避免反馈失真、控制错误和主体表达受损？
- 系统故障时如何降级、恢复和保护主体尊严？
- 如何定义主体连续性、可表达性、可退出性和权利边界？

## 不属于本子域

- 普通长寿干预证据页。
- 医疗建议或手术建议。
- 人体或动物实验操作协议。
- 绕过伦理审查的可执行技术路线。

## 与其他子域的关系

- 与 `longevity-evidence` 的关系：共享证据追溯和风险边界原则，但研究尺度不同。
- 与 `memory-editing` 的关系：共享主体连续性和神经安全问题；本子域关注生命支持与接口，记忆编辑关注记忆痕迹读写。
