# Risk Engineering

Risk Engineering 负责整理事故、灾害、暴力、环境风险、系统故障、尾部风险和恢复工程。

核心问题：

> 主体持续性不仅会被衰老终止，也会被事故、灾害、暴力、制度失败和尾部风险突然截断。

## 先验位置

```text
有效永生 / 主体持续性最大化
  -> 主体必须穿越长时间尺度
  -> 时间越长，遭遇事故、灾害和系统性风险的机会越多
  -> 单次尾部事件足以终止全部未来选择权
  -> 因此风险工程是压低死亡风险函数的基础域
```

## 关注对象

- 事故预防、灾害韧性、公共预警、应急响应和灾后恢复。
- 暴力、战争、犯罪、家庭暴力、极端环境和职业风险。
- 气候风险、空气、水、热、洪水、火灾和基础设施中断。
- 系统安全、冗余、失效模式、恢复时间和个人 / 社区韧性。

## Human Infra 模型链路

```text
风险工程措施 T
  -> 改变风险暴露、预警、避险、响应和恢复能力 X
  -> 改变主体外部安全状态 S
  -> 降低事故、灾害、暴力和系统失效导致的死亡或失能风险 λ(t)
  -> 增加未来选择权和长期行动窗口
```

## 非目标

- 不提供规避法律、攻击、武器、暴力或危险操作指南。
- 不把风险工程简化为个人责任；环境、制度和系统设计同样是对象。
- 不把单点安全建议写成完整风险治理。
