# risk-engineering 目录说明

`domains/risk-engineering/` 保存风险工程研究域。它关注事故、灾害、暴力、系统故障和尾部风险如何改变主体持续性。

## 目录结构

```text
risk-engineering/
├── AGENTS.md
└── README.md
```

## 职责边界

- 本域负责风险暴露、预警、避险、响应、恢复和尾部风险治理。
- 本域可整理灾害、事故、环境风险、公共预警和韧性系统资料。
- 本域不提供攻击、违法、武器、暴力或危险操作指南。

## 维护规则

- 任何风险降低主张必须说明风险源、暴露路径、干预点、失败模式和恢复路径。
- 涉及高风险行为时只做防护、治理和非操作性安全边界。
- 新增文件或子目录时，同步更新本文件。
