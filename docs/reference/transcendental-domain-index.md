# 先验追问研究域索引

本文件把 Human Infra 的研究域从“散点目录”收束为一条先验追问链：不先问某个技术是否有效，而是先问“有效永生 / 主体持续性最大化要成立，背后必须已经具备哪些条件”。

它不是替代 `research-domain-atlas.md`。Atlas 负责生成规则和全量地图；本文件负责把终极目标下的核心路径压缩成可导航的第一性目录。

## 第一性问题

```text
有效永生何以可能？
  -> 必须存在一个可持续行动、学习、修复、选择和进入未来的主体
  -> 主体不是抽象灵魂，而是由身体、认知、记忆、时间、资源、环境、工具和社会支持共同维持的系统
  -> 因而每一个能维护、延展、转移、暂停、放大或保护这个系统的条件，都可以成为 Human Infra 研究域
```

最小判定：

```text
候选研究域 D
  -> 是否改变主体持续性变量 X
  -> 是否改变状态 S 或状态转移 P(S_next | S, A)
  -> 是否改变风险函数 lambda(t)、有效时间、主观时间、相对时间或未来选择权
  -> 是否需要独立证据、失败模式、伦理边界和维护目录
  -> 若是，则进入 domains/；若否，则归入已有域或 source note
```

## 与研究域 Atlas 的关系

`research-domain-atlas.md` 维护全量域地图和域判定契约。本文件维护压缩版主干，解决三个问题：

- 给新读者一个从终极目标进入域目录的最短路径。
- 防止把“山中因子”“黑洞等待室”“AI 增强”等单点热点误认为项目全部。
- 判断新增域是否属于核心主干、分支扩展，还是应回到已有域内部整理。

## 目录维护规则

1. 新增候选域前，先从本文件判断它落在哪个先验节点下。
2. 若已有域能承载，不新建同义目录，只补该域 README、Source Card 或子章节。
3. 若候选对象改变了不同变量、需要不同证据体系、存在不同失败模式和伦理边界，再进入 `domains/`。
4. 新增正式域必须同步更新 `research-domain-atlas.md`、`domain-map.md`、`research-domain-radar.md`、`ethics-and-safety-boundaries.md`、`docs/README.md`、`docs/AGENTS.md`、`README.md` 和仓库检查脚本。
5. 任何“有效永生路径”都必须同时接受证据、风险、可及性、治理、主体连续性和非目标边界审查。

## 当前缺口记录

这些不是立即建目录的任务，而是后续 Source Card 和域内补充优先级：

- `cellular-reprogramming/` 需要继续把 OSKM、OSK、MPTR、mRNA、化学重编程、AI 因子设计和表观遗传编辑拆成 Source Cards。
- `future-waiting/` 需要继续把度规红移递归等待假设中的 `K`、`NCG-Screen`、退出门、连续性门和 synthetic scenarios 固化为可审查模板。
- `cognitive-augmentation/` 需要把 AI 增强与有效永生飞轮中的学习轮次、能力复利、工具共生和主体控制权建立变量接口。
- `resource-social-infra/` 需要把稀缺问题、反稀缺工程和资源预算增量写成价值视角与应用案例。
- `brain-preservation-connectomics-emulation/`、`memory-editing/` 和 `disembodied-cns/` 需要共享主体连续性判据，避免把结构保存、记忆操控和中枢维持混成一个概念桶。
