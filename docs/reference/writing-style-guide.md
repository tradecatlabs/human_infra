# Writing Style Guide

Human Infra documentation should be precise, traceable, and safety-aware.

## Voice

- Use direct language.
- Prefer short paragraphs.
- Define terms before relying on them.
- Avoid hype, certainty theater, and moralizing.

## Evidence

- Separate evidence from interpretation.
- Preserve uncertainty.
- Link conclusions to sources when possible.
- Do not present mechanism speculation as proven effect.

## Safety

- Do not provide medical diagnosis or treatment instructions.
- Do not provide invasive neural, memory manipulation, or human experimentation procedures.
- Do not frame people as resources to optimize without consent, dignity, and recovery boundaries.

## Structure

- One document should have one primary job.
- Put stable lookup material in `docs/reference/`.
- Put conceptual essays in `docs/explanations/`.
- Put procedures in `docs/how-to/`.
- Put raw inputs in `docs/source-notes/`.
- Update directory `README.md` and `AGENTS.md` when navigation changes.
