# How-To Guides

How-to documents are task-oriented procedures. They should be short, practical, and focused on one job.

Current documents:

- `add-domain.md`: how to add a new Human Infra domain.
- `add-source-note.md`: how to add raw or lightly processed source input.
- `contribute-docs.md`: how to add or update repository documentation.
- `run-quality-checks.md`: how to run local repository checks.
- `share-human-infra.md`: how to introduce Human Infra in public posts, community messages, and launch threads.
